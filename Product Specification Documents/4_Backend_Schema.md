# Backend & Database Schema
**Product Name:** Google Rhythm  
**Version:** 1.0  
**Storage Type:** Client-Side IndexedDB (via Dexie.js)

*Note: Google Rhythm operates on a "Local-First" architecture. There is no remote PostgreSQL/MongoDB backend. This schema defines the structure of the local IndexedDB.*

## 1. Dexie Database Definition
```javascript
const db = new Dexie('GoogleRhythmDB');
db.version(1).stores({
  logs: 'date, cycleDay, phase, symptoms, mood, flow, notes',
  insightsCache: 'date, text, generatedAt'
});
```

## 2. Table: `logs`
This table stores the user's daily health and symptom logs.

| Field | Type | Primary Key / Index | Description |
| :--- | :--- | :--- | :--- |
| `date` | `String` | **Primary Key** | The ISO date string (YYYY-MM-DD) representing the day of the log. Guarantees one entry per day. |
| `cycleDay` | `Number` | | The numeric day of the user's cycle (e.g., 1 to 28) on this specific date. |
| `phase` | `String` | Indexed | The calculated biological phase for this date (e.g., 'MENSTRUAL', 'FOLLICULAR'). |
| `symptoms` | `Array<String>` | | Array of symptom tags selected by the user (e.g., `["Cramps", "Headache"]`). |
| `mood` | `String` | | The user's primary selected mood for the day. |
| `flow` | `String` | | For menstrual days, the level of bleeding (e.g., `Light`, `Medium`, `Heavy`). |
| `notes` | `String` | | Optional free-text diary entry. |

## 3. Table: `insightsCache`
This table acts as a rate-limiting and performance cache for Gemini AI responses.

| Field | Type | Primary Key / Index | Description |
| :--- | :--- | :--- | :--- |
| `date` | `String` | **Primary Key** | The ISO date string (YYYY-MM-DD) the insight was generated for. |
| `text` | `String` | | The markdown-formatted response string generated by the Gemini API. |
| `generatedAt` | `Number` | | Unix timestamp indicating exactly when the API call was completed. |

## 4. Zustand Store Structure (`userPrefs`)
*Stored in `localStorage` via Zustand Persist Middleware. This acts as the "User Profile" table.*

```json
{
  "userPrefs": {
    "isOnboardingComplete": true,
    "name": "Sarah",
    "dob": "1990-05-15",
    "lifecycleMode": "cycle",
    "cycleLength": 28,
    "lastPeriodDate": "2026-06-01",
    "contraceptive": "None",
    "diagnosedConditions": ["PCOS"],
    "dietPreference": "vegetarian",
    "activityLevel": "Moderate",
    "aiTone": "Warm & Supportive",
    "isCalendarSynced": true,
    "isPartnerSynced": false
  }
}
```

## 5. Cloud Serialization (`google_rhythm_backup.json`)
When the user connects Google Drive, the above local tables are serialized into a single JSON payload.

```json
{
  "version": 1,
  "timestamp": "2026-06-09T18:00:00Z",
  "data": {
    "logs": [ ...array of log objects... ],
    "insightsCache": [ ...array of insight objects... ]
  }
}
```
