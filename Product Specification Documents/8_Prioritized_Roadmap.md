# Google Rhythm: Prioritized Feature Roadmap & Build Plan

## Prioritization Framework
Every feature is scored across 3 dimensions:
- **Impact:** How much does this improve user experience or competitive position? (1–5)
- **Effort:** How complex is it to build? (1=easy, 5=very hard)
- **Uniqueness:** Does any competitor have this? (🔴 None | 🟠 Partial | 🟢 Everyone has it)

## 🏁 Decision: Start With BUILD 1 — Onboarding Wizard

**Why the Onboarding Wizard MUST come first:**
1. It collects the user's name, DOB, conditions (PCOS, Endo, PMDD), diet, and cycle data.
2. Every single feature below it depends on this data. Without it, the AI has no context, the PCOS mode has no trigger, the skincare card has no phase info, the doctor's report has no history.
3. It's the single biggest thing separating Google Rhythm from "just another demo" to a real, living product.
4. The 6-step wizard is mostly UI — it stores data in Zustand (already extended for all these fields) and redirects to the dashboard when done.
